📝 Three-Address Code

Translates complex expressions into a sequence of simple instructions, each having at most three operands — two sources and one destination using temporaries.

📊 Quadruples

Represents each TAC instruction as a 4-field record: (operator, arg1, arg2, result). This tabular form makes optimization and code generation straightforward.

📋 Triples

An alternative to quadruples that removes the explicit result field. Instead, each instruction is referenced by its index number, reducing storage requirements.

🔄 Temporary Variables

The ICG creates temporary variables (t1, t2, t3...) to hold intermediate computation results when complex expressions are decomposed into simple operations.

🏷️ Label Generation

For control flow statements (if-else, while, for), the ICG generates labels (L0, L1, L2...) and conditional/unconditional goto instructions.

🎯 Machine Independence

Intermediate code is independent of the target machine architecture, enabling the same front-end to generate code for different back-end targets.

Theory

Understanding the fundamentals of Intermediate Code Generation

What is an Intermediate Code Generator?

The Intermediate Code Generator (ICG) is the fourth phase of a compiler. After lexical analysis, parsing, and semantic analysis, the ICG translates the validated Abstract Syntax Tree (AST) into an Intermediate Representation (IR). This IR is a low-level, machine-independent code that is easier to optimize and translate into actual machine code.


Key Functions

  • Expression Decomposition — breaks complex expressions into simple three-address instructions
  • Temporary Generation — creates temporary variables (t1, t2, ...) for intermediate results
  • Label Generation — creates labels (L0, L1, ...) for control flow targets
  • Control Flow Translation — converts if-else, while, for into goto-based instructions
  • Assignment Translation — converts assignments into TAC form
  • Quadruple/Triple Generation — produces tabular IR representations

How It Works

The ICG traverses the AST in a specific order. For each node:

  • If it's an arithmetic expression like a + b * c, it decomposes into: t1 = b * c then t2 = a + t1
  • If it's an assignment like x = a + b * c;, it adds: x = t2
  • If it's an if statement, it generates conditional jumps: if a > b goto L0, goto L1
  • If it's a while loop, it generates labels and back-edges for looping

Example

Source code:

  x = a + b * c - d / e;

Three-Address Code (TAC):

  t1 = b * c
  t2 = d / e
  t3 = a + t1
  t4 = t3 - t2
  x  = t4

Three-Address Code

The most common intermediate representation form

What is Three-Address Code?

Three-Address Code (TAC) is a sequence of instructions where each instruction has at most three addresses — typically two operand addresses and one result address. 

  result = operand1  op  operand2
